# 项目介绍

Creator Data Tracker 是一个全平台运营数据仪表盘，自动采集多平台创作者数据和网站流量数据，部署到 Vercel 后团队可随时在线查看。

## 页面组成

仪表盘由以下几个区域组成：

### 1. 平台数据卡片（左侧）

每个平台一张卡片，展示该平台的核心数据：

| 指标 | 说明 |
|------|------|
| 作品 | 已发布作品总数 |
| 粉丝 | 当前粉丝数 |
| 播放 | 作品总播放量 |
| 点赞 | 作品总点赞数 |
| 评论 | 作品总评论数 |
| 分享 | 作品总分享数 |
| 收藏 | 作品总收藏数 |
| 总互动 | 以上所有指标之和 |

每个指标下方显示 **增量**（vs 昨天 / 7天前 / 30天前，可切换）。

**支持的平台：**
- 抖音（creator.douyin.com）
- 小红书（creator.xiaohongshu.com）
- 视频号（channels.weixin.qq.com）

**数据来源：** `data/all_data.json` → `daily_snapshots` 数组
**数据获取：** `collect_all.py` 通过 Playwright 浏览器自动化，用平台 Cookie 登录后抓取 API 数据

---

### 2. 官网数据监测（右上）

展示 Google Analytics 4 网站流量数据，有三个 tab 页：

#### 全部 tab
| 指标 | 说明 |
|------|------|
| 注册人数（昨天） | 昨日新注册用户数 |
| UV（独立访客） | 独立访问用户数 |
| PV（页面浏览） | 页面浏览总量 |
| 会话数 | 用户会话数（一次访问=一个会话） |
| 平均时长 | 平均每次会话停留时间 |

#### 前台官网 tab
只看前台网站（如 example.com）的数据，并展示 TOP 3 热门页面。

#### 后台系统 tab
只看后台应用（如 app.example.com）的数据，并展示 TOP 3 热门页面。

**数据来源：** `data/ga_data.json`
- 全部 → `overview_1d` / `overview_7d` / `overview_30d`
- 前台 → `segmented_1d.frontend` / `segmented_7d.frontend` / `segmented_30d.frontend`
- 后台 → `segmented_1d.backend` / `segmented_7d.backend` / `segmented_30d.backend`

**数据获取：** `scripts/collect_ga.py` 通过 Google Analytics Data API，用服务账号凭证查询。域名过滤通过 `ALLOWED_HOSTNAMES`、`FRONTEND_HOSTNAMES`、`BACKEND_HOSTNAMES` 配置。

---

### 3. 订单与收入栏

| 指标 | 说明 |
|------|------|
| 累计订单 | 总付费订单数 |
| +N 新增 | 今日新增订单数 |
| 累计金额 | 总收入金额（USD） |

**数据来源：** `data/orders_data.json` → `total_paid_orders`、`total_revenue_usd`、`daily` 数组
**数据获取：** 这个文件不由采集脚本生成。需要通过以下方式之一推送到仓库：
- **AI 机器人共创者**：将机器人添加为仓库 Collaborator，机器人定时从后台 API 拉取数据并 push
- **自己写脚本**：调用后台 API 生成 JSON 文件后 git push
- **手动更新**：直接编辑 JSON 文件

> 不需要订单数据？不创建这个文件，仪表盘自动隐藏此区域。

---

### 4. 趋势图表

图表区域有四个 tab 可切换：

| Tab | 展示内容 | 数据来源 |
|-----|----------|----------|
| PV/UV | 每日页面浏览量和独立访客趋势线 | `ga_data.json` → `daily_trend` |
| 注册趋势 | 每日注册用户数柱状图 | `ga_data.json` → `signup_trend` 或 `registration_data.json` |
| 下单趋势 | 每日订单数和收入趋势 | `orders_data.json` → `daily` |
| 收起 | 收起图表区域 | — |

---

### 5. 流量明细表格（下方）

分为多个小表格：

| 表格 | 展示内容 | 数据来源 |
|------|----------|----------|
| 热门页面 | 浏览量最高的页面路径 | `ga_data.json` → `top_pages` |
| 流量来源 | 访问来源/媒介和会话数 | `ga_data.json` → `traffic_sources` |
| 着陆页 | 用户进入网站的第一个页面 | `ga_data.json` → `landing_pages` |
| 退出页 | 用户离开网站的最后一个页面 | `ga_data.json` → `exit_pages` |
| 注册来源转化率排行 | 各来源的注册数和转化率 | `ga_data.json` → `signup_by_source_*` |

---

### 6. 底部统计表

| 表格 | 展示内容 | 数据来源 |
|------|----------|----------|
| 设备类型 | 移动端/桌面端/平板占比 | `ga_data.json` → `devices` |
| 操作系统 | iOS/Android/Windows 等 | `ga_data.json` → `operating_systems` |
| 语言 | 用户浏览器语言 | `ga_data.json` → `languages` |
| 地理分布 | 用户所在国家/地区 | `ga_data.json` → `geo` |

## 数据文件总览

| 文件 | 生成方式 | 内容 | 是否必需 |
|------|----------|------|----------|
| `data/all_data.json` | `collect_all.py` 自动生成 | 平台数据（粉丝、播放、点赞等） | 是（至少一个平台） |
| `data/ga_data.json` | `scripts/collect_ga.py` 自动生成 | GA4 网站流量数据 | 可选 |
| `data/orders_data.json` | 后台 API / 手动推送 | 订单数量和收入金额 | 可选 |
| `data/registration_data.json` | 后台 API / 手动推送 | 每日注册用户数 | 可选 |
| `data/access_code.json` | 手动创建 | 仪表盘访问密码 | 是 |

## 数据获取方式汇总

| 数据 | 获取方式 | 自动化程度 | 配置文档 |
|------|----------|-----------|----------|
| 抖音数据 | Cookie 登录 + API 抓取 | 全自动（Cookie 有效期约 14 天） | [平台配置](配置文档.md) |
| 小红书数据 | Cookie 登录 + API 抓取 | 全自动（Cookie 有效期约 14 天） | [平台配置](配置文档.md) |
| 视频号数据 | 微信扫码登录 + API 抓取 | **每天需扫码一次**（Cookie 仅几小时有效） | [平台配置](配置文档.md) |
| GA 网站数据 | Google Analytics Data API | 全自动 | [GA 配置指南](GA配置指南.md) |
| 订单/收入数据 | 后台 API 推送到仓库 | 取决于对接方式 | [后台数据对接](后台数据对接.md) |
| 注册用户数据 | 后台 API 推送到仓库 | 取决于对接方式 | [后台数据对接](后台数据对接.md) |

## 自动化流程

```
每天定时触发（macOS launchd）
    │
    ├── Step 1: 采集抖音/小红书（全自动）
    │
    ├── Step 2: 采集视频号（弹窗扫码）
    │
    ├── Step 3: 采集 GA 数据（全自动）
    │
    ├── Step 4: 数据写入 data/*.json + SQLite
    │
    └── Step 5: git push → Vercel 自动部署 → 仪表盘更新
```

后台订单/注册数据由另外的流程（机器人或脚本）推送到仓库，同样触发 Vercel 部署。

## 仪表盘访问

- 部署到 Vercel 后，访问你的 `.vercel.app` 域名
- 需要输入 6 位访问码（存储在 `data/access_code.json` 中）
- 访问码解锁后保存在浏览器 sessionStorage，关闭标签页后需重新输入
