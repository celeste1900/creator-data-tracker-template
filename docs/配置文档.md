# 创作者数据追踪系统 - 配置文档

## 切换监测账号

只需更换对应平台的 Cookie，即可切换到其他账号进行数据监测。

## 配置文件位置

```
项目根目录/config.json
```

## 抖音配置

### 获取 Cookie

1. 浏览器打开 https://creator.douyin.com/ 并登录目标账号
2. 按 `F12` 打开开发者工具
3. 切换到 `Network` 标签
4. 刷新页面
5. 点击任意请求，在 `Headers` 中找到 `Cookie`
6. 复制完整的 Cookie 值

### 更新配置

编辑 `config.json`：
```json
{
  "douyin": {
    "enabled": true,
    "cookie": "粘贴你的Cookie到这里"
  }
}
```

## 小红书配置

### 获取 Cookie 和 User ID

1. 浏览器打开 https://creator.xiaohongshu.com/ 并登录目标账号
2. 按 `F12` 打开开发者工具
3. 切换到 `Network` 标签
4. 刷新页面
5. 找到 `Cookie` 并复制
6. 在 Cookie 中找到 `x-user-id-creator.xiaohongshu.com=xxx`，复制 `xxx` 部分作为 User ID

### 更新配置

编辑 `config.json`：
```json
{
  "xiaohongshu": {
    "enabled": true,
    "user_id": "你的User ID",
    "cookie": "粘贴你的Cookie到这里"
  }
}
```

## 视频号配置

> **重要说明**：视频号采集需要每天扫码登录，无法完全自动化。
>
> 采集脚本会在定时任务触发时自动弹出浏览器窗口，你需要在 **2 分钟内** 用微信扫码登录。
> macOS 会弹出系统通知提醒你扫码。

### 工作原理

1. 定时任务触发采集（如每天 9:15）
2. 脚本自动打开浏览器，显示视频号助手登录页
3. macOS 通知提醒："请扫码登录视频号（2分钟内）"
4. 你用微信扫码确认登录
5. 登录成功后，脚本自动采集数据（包括数据中心的实时互动数据）
6. 采集完成，浏览器自动关闭

### 配置说明

```json
{
  "shipinhao": {
    "enabled": true,
    "cookie": "",
    "cookie_expires_hint": 4
  }
}
```

> Cookie 字段可以留空——脚本检测到 Cookie 失效后会自动弹窗扫码登录。
> 登录状态保存在 `data/browser_data/shipinhao/` 目录中。

### 注意事项

- **定时任务时间选择**：建议设在你确定在电脑前的时间（如早上 9:00-10:00）
- **错过扫码**：如果 2 分钟内没有扫码，视频号采集会跳过，其他平台不受影响
- **节假日**：不在电脑前时，视频号数据会暂停采集，恢复后继续

## 完整配置示例

```json
{
  "douyin": {

    "github_repo": "your-username/creator-data-tracker"
  }
}
```

## 定时任务配置

### 设置每日自动采集

运行定时任务管理脚本：

```bash
python scripts/setup_cron.py
```

按提示选择 **1. 安装定时任务**，然后输入采集时间：

```
每天什么时候执行数据采集？
小时 (0-23, 默认 9): 9
分钟 (0-59, 默认 0): 15
```

> **时间选择建议**：如果启用了视频号采集，请选择你确定在电脑前的时间。脚本会弹窗提醒你扫码。

### 管理定时任务

```bash
python scripts/setup_cron.py
# 选择 1-安装 / 2-卸载 / 3-查看状态 / 4-立即执行一次
```

### 定时任务执行流程

```
定时触发 → 采集抖音/小红书（自动） → 采集GA数据（自动）
         → 弹窗扫码 → 采集视频号（需扫码）
         → 数据导出 → 自动 git push
```

### 日志查看

```bash
# 查看采集日志
cat logs/collect.log

# 查看错误日志
cat logs/collect.error.log
```

## 其他设置

| 配置项 | 说明 |
|--------|------|
| `works_limit` | 每个平台最多采集的作品数量 |
| `auto_push_to_github` | 采集后是否自动推送到 GitHub |
| `github_repo` | GitHub 仓库地址 |

---

## 更新配置后

1. 保存 `config.json`
2. 手动运行一次采集验证配置是否正确：
   ```bash
   python collect_all.py
   ```
3. 如果成功，之后会按定时任务自动采集
