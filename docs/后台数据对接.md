# 后台数据对接

仪表盘支持展示你的后台业务数据：订单金额、注册用户、转化率等。这个模块是 **可选的**，不配置也不影响平台数据采集和 GA 面板。

---

## 原理

仪表盘读取仓库中的两个 JSON 文件来展示后台数据：

- `data/orders_data.json` → 订单数、收入金额、累计趋势
- `data/registration_data.json` → 每日新注册、累计用户、转化率

这两个文件 **不是** 由采集脚本自动生成的，需要你通过某种方式把后台数据推送到仓库中。

---

## 数据格式

### 订单数据 — `data/orders_data.json`

```json
{
  "updated_at": "2026-03-17T10:00:00+08:00",
  "total_paid_orders": 280,
  "total_revenue_usd": 8500,
  "daily": [
    {
      "date": "2026-03-15",
      "orders": 12,
      "revenue_usd": 360,
      "cumulative_orders": 268,
      "cumulative_revenue_usd": 8140
    },
    {
      "date": "2026-03-16",
      "orders": 12,
      "revenue_usd": 360,
      "cumulative_orders": 280,
      "cumulative_revenue_usd": 8500
    }
  ]
}
```

| 字段 | 说明 |
|------|------|
| `orders` | 当日新增付费订单数 |
| `revenue_usd` | 当日收入（美元） |
| `cumulative_orders` | 截至当日的累计订单数 |
| `cumulative_revenue_usd` | 截至当日的累计收入 |

### 注册数据 — `data/registration_data.json`

```json
{
  "updated_at": "2026-03-17T10:00:00+08:00",
  "total_users": 6200,
  "daily": [
    { "date": "2026-03-15", "count": 180 },
    { "date": "2026-03-16", "count": 210 }
  ]
}
```

| 字段 | 说明 |
|------|------|
| `count` | 当日新注册用户数 |
| `total_users` | 总注册用户数 |

## 推送方式

你需要选择一种方式把数据推送到仓库。以下是几种常见方案：

### 方案一：AI 机器人共创者（推荐）

将一个 AI 代码机器人（如 OpenClaw、GitHub Bot）添加为仓库共创者（Collaborator），让它定时从你的后台 API 拉取数据并推送到仓库。

**流程：**
1. 在 GitHub 仓库 Settings → Collaborators 中邀请机器人账号
2. 机器人定时调用你的后台 API 获取订单/注册数据
3. 机器人将数据写入 `data/orders_data.json` 和 `data/registration_data.json`
4. 机器人 git commit + push 到仓库
5. Vercel 自动部署，仪表盘更新

**优点**：全自动，不需要你手动操作。

### 方案二：自己写脚本

写一个脚本从你的后台 API 拉取数据，输出为上述 JSON 格式，然后 git push。

```bash
# 示例：拉取数据并推送
python scripts/fetch_backend_data.py  # 你自己写的脚本
git add data/orders_data.json data/registration_data.json
git commit -m "Update backend data"
git push
```

可以把这个脚本加入定时任务，和平台采集一起运行。

### 方案三：手动更新

直接编辑 JSON 文件，适合数据量小或更新频率低的场景。

---

## 仪表盘展示效果

接入后台数据后，仪表盘会展示：

- 💰 今日/累计订单数和收入金额
- 📈 每日订单和收入趋势图
- 👤 每日新注册用户数
- 📊 转化率（订单数 ÷ 注册数）
- 📉 注册趋势和累计用户增长

---

## 不需要后台数据？

完全没问题。不创建这两个文件，仪表盘会自动隐藏订单和注册相关的模块，只显示平台数据和 GA 数据。
